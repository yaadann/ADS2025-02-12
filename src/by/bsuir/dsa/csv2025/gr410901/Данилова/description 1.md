# Название

Сортировка продаж

# Описание

Сортировка продаж

## Условие задачи

Компания занимается анализом данных о продажах. У вас есть информация о заказах, каждый из которых принадлежит определенной категории товаров. Необходимо обработать серию запросов, определяющих для каждой категории количество заказов, сумма которых превышает заданное значение.

### Формат входных данных

Первая строка содержит два целых числа:
- `n` (1 ≤ n ≤ 10^5) - количество заказов
- `m` (1 ≤ m ≤ 10^3) - количество запросов

Следующие `n` строк описывают заказы в формате:
- `id` - уникальный идентификатор заказа
- `amount` (1 ≤ amount ≤ 10^9) - сумма заказа
- `category` (1 ≤ category ≤ 10^5) - категория товара

Последние `m` строк содержат пороговые значения для запросов.

### Формат выходных данных

Для каждого из `m` запросов выведите в отдельной строке через пробел количество заказов в каждой категории (в порядке возрастания номеров категорий), сумма которых превышает пороговое значение.

### Пример

**Входные данные:**

3 2

1 100 1

2 200 1

3 150 2

100

200

**Выходные данные:**

1 1

0 0

### Ограничения

- Время работы: O(n log n + m log n)
- Память: O(n)
- Гарантируется уникальность идентификаторов заказов

# Сортировка


**Сортировка** — один из фундаментальных алгоритмических примитивов, заключающийся в упорядочивании элементов множества согласно заданному критерию. Эффективные алгоритмы сортировки лежат в основе многих задач обработки данных, поиска и анализа.

## Основные понятия и терминология

### Ключевые определения

- **Ключ сортировки** — значение, по которому происходит сравнение элементов
- **Стабильность** — свойство алгоритма сохранять относительный порядок элементов с одинаковыми ключами
- **Адаптивность** — алгоритм эффективно работает с частично отсортированными данными

## Классификация алгоритмов сортировки

### Сортировка пузырьком (Bubble Sort)

#### Принцип работы
Алгоритм многократно проходит через массив, сравнивая соседние элементы и меняя их местами, если они находятся в неправильном порядке. За каждый проход наибольший элемент "всплывает" в конец массива.

### Сортировка выбором (Selection Sort)

#### Принцип работы
Алгоритм на каждом шаге находит минимальный элемент в неотсортированной части массива и помещает его в конец отсортированной части.

### Сортировка вставками (Insertion Sort)

#### Принцип работы
Алгоритм строит отсортированную последовательность, по одному элементу за раз, вставляя каждый новый элемент в правильную позицию.

### Быстрая сортировка (QuickSort)

#### Принцип работы
Выбор опорного элемента (pivot).
Разделение массива на элементы меньше, равные и больше pivot.
Рекурсивная сортировка подмассивов

### Сортировка слиянием (MergeSort)

#### Принцип работы
Алгоритм рекурсивно разделяет массив пополам, сортирует каждую половину и затем сливает их в один отсортированный массив.

### Пирамидальная сортировка (HeapSort)

#### Принцип работы
Алгоритм строит max-heap из массива, затем многократно извлекает максимальный элемент и перестраивает кучу.
