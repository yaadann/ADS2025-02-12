Сортировка вставками - реализация и тестирование

# Описание

Задание: Сортировка вставками - реализация и тестирование

Создайте класс InsertionSort с методом main.
Реализуйте алгоритм сортировки вставками для целочисленных массивов.

С консоли вводится:
- Количество элементов N
- N целых чисел через пробел

Программа должна:
1. Вывести исходный массив
2. Вывести все промежуточные состояния массива после каждой итерации внешнего цикла
3. Вывести отсортированный массив

Алгоритм сортировки вставками:
for i от 1 до n-1:
key = arr[i]
j = i - 1
Пока j >= 0 и arr[j] > key:
arr[j + 1] = arr[j]
j = j - 1
arr[j + 1] = key

Пример:
Ввод:
6
5 2 4 6 1 3

Вывод:
Исходный массив: [5, 2, 4, 6, 1, 3]
После итерации 1: [2, 5, 4, 6, 1, 3]
После итерации 2: [2, 4, 5, 6, 1, 3]
После итерации 3: [2, 4, 5, 6, 1, 3]
После итерации 4: [1, 2, 4, 5, 6, 3]
После итерации 5: [1, 2, 3, 4, 5, 6]
Отсортированный массив: [1, 2, 3, 4, 5, 6]

Сортировка вставками — один из простейших алгоритмов сортировки, который строит отсортированный массив по одному элементу за раз. Хотя он неэффективен для больших наборов данных, он превосходно работает в определенных сценариях и служит отличным учебным инструментом для понимания фундаментальных концепций сортировки.

## Сложность алгоритма

Временная сложность сортировки вставками раскрывает ее сильные и слабые стороны:

- **Лучший случай:** $O(n)$ - когда массив уже отсортирован
- **Средний случай:** $O(n^2)$ - для случайных входных данных
- **Худший случай:** $O(n^2)$ - когда массив отсортирован в обратном порядке
- **Пространственная сложность:** $O(1)$ - алгоритм сортировки на месте

Квадратичная сложность возникает из-за вложенной структуры циклов, где каждый элемент может потребовать сравнения со всеми предыдущими элементами в отсортированной части.

## Адаптивная природа

Сортировка вставками *адаптивна* — она становится более эффективной при обработке частично отсортированных массивов. Это свойство делает ее практичной для реальных сценариев, где данные часто поступают в почти отсортированном порядке.

Производительность алгоритма можно анализировать через подсчет операций:

- **Сравнения:** Элементы сравниваются для нахождения правильной позиции вставки
- **Перестановки:** Элементы сдвигаются вправо для освобождения места под вставку

Для массива размера $n$ максимальное количество сравнений и перестановок составляет:

$$\sum_{i=1}^{n-1} i = \frac{n(n-1)}{2} = O(n^2)$$

## Практические соображения

**Когда использовать сортировку вставками:**
- Малые наборы данных ($n < 50$)
- Почти отсортированные данные
- Сценарии онлайн-сортировки
- Образовательные цели
- Как строительный блок в более сложных алгоритмах (например, Timsort)

**Преимущества:**
- Простая реализация
- Стабильная сортировка
- Работа на месте
- Адаптивная производительность
- Низкие накладные расходы памяти

**Математическое понимание:**

Ожидаемое количество сравнений для случайных входных данных составляет примерно $\frac{n^2}{4}$, что вдвое меньше худшего случая. Эта ожидаемая производительность в сочетании с поведением $O(n)$ в лучшем случае делает сортировку вставками удивительно эффективной для многих практических применений, несмотря на ее сложность $O(n^2)$ в худшем случае.

## Образовательная ценность

Изучение сортировки вставками дает фундаментальное понимание:
- Алгоритмического мышления и декомпозиции задач
- Анализа временной сложности
- Компромиссов между простотой и эффективностью
- Проектирования адаптивных алгоритмов
- Эмпирического измерения производительности через подсчет операций

Алгоритм служит введением в понимание более сложных стратегий сортировки и остается актуальным в современных вычислениях, несмотря на свою простоту.
