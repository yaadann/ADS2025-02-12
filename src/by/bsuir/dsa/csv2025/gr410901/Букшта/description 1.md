# Название

Задача: Сжатие координат

# Описание

Условие задачи 
Компания ведёт сбор GPS-координат курьеров в течение дня. Каждая координата представлена целыми числами `(x, y)`.
За день может быть собрано до `10^5` координат.
Для оптимизации хранения данных нужно заменить каждую координату на её **ранговый номер** в списке 
всех уникальных координат по оси `x` и по оси `y` (то есть выполнить сжатие координат).  

**Входные данные:**  
- Первое число `n` — количество координат.  
- Далее `n` строк с двумя числами `x_i` и `y_i`.  

**Выходные данные:**  
- `n` строк с координатами `(x'_i, y'_i)` после сжатия.  
- Координаты должны быть заменены на числа от `0` до `k-1` (где `k` — количество уникальных значений по соответствующей оси).  

**Пример:**

Вход:

5
100 200
50 300
100 300
50 200
200 100


Выход:

2 1
0 2
2 2
0 1
3 0

# Краткая теория

Теория: Сжатие координат

**Сжатие координат** — это замена исходных значений на их ранги среди  
отсортированных уникальных координат.  
Для множества значений по оси выполняется сортировка уникальных элементов, после чего  
каждому значению присваивается номер его позиции.

Пусть заданы координаты:
$$
(x_i, y_i),\quad i = 1..n
$$

После сжатия каждая координата преобразуется в:
$$
(x'_i, y'_i) = (\mathrm{rankX}(x_i),\ \mathrm{rankY}(y_i))
$$

где `rankX` и `rankY` — индексы значений в отсортированных массивах уникальных `x` и `y`.

Сложность выполнения:
$$
O(n \log n)
$$
