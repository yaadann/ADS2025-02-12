# Название

Определение изолированных групп в социальных сетях

# Описание

Определение изолированных групп в социальных сетях
В социальной сети каждая вершина графа — это пользователь, а каждое ребро — дружеская связь между ними.  
Необходимо определить, сколько изолированных групп пользователей существует, и перечислить участников каждой группы.  
Если два пользователя связаны напрямую или через других пользователей, они относятся к одной компоненте связности.

### **Формат ввода**

Первая строка содержит два числа: количество пользователей (вершин) и количество связей (рёбер).  
Далее идут строки с номерами пользователей (0..n-1), которые дружат друг с другом.

### **Формат вывода**

Сначала выводится число компонент связности.  
Затем для каждой компоненты выводится список номеров пользователей, входящих в неё.


# Краткая теория

Поиск компонент связности

Компонента связности в графе — это максимальное множество вершин, таких что любая пара соединена путем по рёбрам, а вершины вне компоненты недостижимы из неё. Для графа  
$$
G = (V, E)
$$
компонентой связности называют подмножество вершин  
$$
C \subseteq V
$$
такое, что каждая вершина $u \in C$ достижима из любой другой $v \in C$, а вершины вне $C$ недостижимы из неё.  

Анализ компонент связности помогает выявлять изолированные группы, исследовать структуру графа и принимать решения при проектировании сетей, кластеризации данных или изучении социальных связей.  

## Основная идея алгоритма

Для нахождения компонент связности используется **обход графа в глубину (DFS)**. Алгоритм выполняется следующим образом:  

1. Все вершины помечаются как непосещённые.  
2. Выбирается непосещённая вершина, от которой запускается DFS. Все вершины, до которых можно дойти из этой точки, включаются в одну компоненту связности.  
3. После завершения обхода отмеченные вершины формируют отдельную компоненту.  
4. Процесс повторяется для оставшихся непосещённых вершин до обработки всего графа.  

Использование DFS позволяет рекурсивно углубляться в граф, эффективно исследовать структурные особенности каждой компоненты и хранить промежуточные результаты в стеке вызовов.  

## Реализация алгоритма

Для хранения графа применяют список смежности adj[v], массив visited[v] для отметки посещённых вершин и массив списков components[i] для хранения вершин каждой компоненты. Такой формат делает работу алгоритма простой и эффективной: доступ к соседям осуществляется быстро, а учёт посещённых вершин позволяет избежать повторных обходов.

Затем алгоритм последовательно просматривает все вершины графа и, встретив непосещённую вершину, запускает обход в глубину, который формирует новую компоненту связности. В ходе DFS каждая достижимая вершина помечается как посещённая и добавляется в текущую компоненту. После завершения обхода компонента сохраняется, и процесс продолжается, пока не будут обработаны все вершины. В итоге граф автоматически разбивается на отдельные изолированные группы.

## Сложность алгоритма

При хранении графа списками смежности сложность обхода составляет  
$$
O(n + m),
$$  
где $n$ — количество вершин, $m$ — количество рёбер. Такой метод оптимален для разреженных графов. Если граф представлен матрицей смежности, сложность увеличивается до  
$$
O(n^2),
$$  
так как необходимо проверять наличие рёбер между каждой парой вершин.  

## Применение компонент связности

Компоненты связности находят применение в социальных сетях для выявления групп пользователей, в инфраструктурных системах для определения подсетей, в географии для выделения изолированных территорий, в биологии для анализа взаимодействий между генами или нейронными сетями, а также в обработке данных и кластеризации объектов.
