# Название

Множество на хеш-таблице

# Описание

Множество на хеш-таблице

Дан набор запросов к множеству целых чисел. Требуется поддерживать следующие операции:

- `add x` — добавить число `x` во множество (если оно уже есть, множество не меняется);
- `del x` — удалить число `x` из множества (если его нет, множество не меняется);
- `find x` — вывести `YES`, если `x` есть во множестве, и `NO` иначе.

Все числа `x` целые, удовлетворяют неравенству `0 ≤ x < 10^9`.

Во входных данных:

- в первой строке задано целое число `q` — количество запросов (`1 ≤ q ≤ 2 * 10^5`);
- в следующих `q` строках записаны запросы в одном из трёх форматов:
  - `add x`
  - `del x`
  - `find x`

Для каждого запроса вида `find` выведите результат в отдельной строке.

Рекомендуется реализовать структуру данных на основе хеш-таблицы (с методом цепочек или открытой адресацией).


# Краткая теория

Хеш-таблица — это структура данных, которая хранит элементы в массиве по индексу, равному значению хеш-функции $h(x)$.

Простой вариант для целых чисел:
$$h(x) = x \bmod m,$$
где $m$ — размер таблицы.

Более устойчивый вариант:
$$h(x) = ((a x + b) \bmod p) \bmod m,$$
где $p$ — большое простое число, а $a, b$ — параметры.

Если разные ключи дают одинаковый хеш (коллизия), её обрабатывают:

- методом цепочек (список в каждой ячейке),
- либо открытой адресацией (поиск следующей свободной позиции).

При хорошем выборе $h(x)$ ожидаемое время операций `add`, `del`, `find` равно $O(1)$.
