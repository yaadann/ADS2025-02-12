# Название

Лабиринт таксиста

# Описание

Лабиринт таксиста

Таксист находится в городе с прямоугольной планировкой, который можно представить как сетку N x M. Некоторые перекрестки (клетки сетки) перекрыты из-за ремонта и являются непроходимыми. Таксист знает свое текущее местоположение (startX, startY) и точку назначения (endX, endY).

# Краткая теория

## Алгоритм Поиска в Ширину (BFS)

### Идея алгоритма
BFS обходит граф слой за слоем. Он начинает из начальной вершины, затем посещает всех её непосредственных соседей, затем соседей соседей и так далее. Для этого используется очередь.

### Почему BFS?
Таксист ищет путь с минимальным количеством кварталов. Каждый ход имеет одинаковую стоимость (1 квартал). BFS гарантированно находит кратчайший путь в невзвешенном графе.

### Алгоритм для лабиринта
1. Создать очередь и добавить стартовую клетку с расстоянием 0
2. Создать массив для отметки посещённых клеток
3. Пока очередь не пуста:
   - Извлечь текущую клетку
   - Если это конечная клетка, вернуть её расстояние
   - Для каждой из четырёх соседних клеток:
     * Если клетка в пределах карты, свободна и не посещена:
       - Пометить её как посещённую
       - Записать расстояние = текущее расстояние + 1
       - Добавить в очередь
4. Если цель не найдена, вернуть -1

### Сложность
Временная сложность: O(N * M)
