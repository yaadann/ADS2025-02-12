# Название

Минимальная максимальная нагрузка

# Описание

Минимальная максимальная нагрузка

## Условие

Даны `n` процессов и `m` одинаковых вычислительных узлов (машин).  
Каждый процесс `i` требует `tasks[i]` единиц времени для выполнения.  

Необходимо распределить все процессы по узлам так, чтобы **максимальная загрузка любого узла была минимальной**.

## Входные данные (одна строка, числа через пробел):

- `n` — количество процессов
- `tasks` — массив целых чисел длины `n`, где `tasks[i]` — время выполнения i-го процесса.  
- `m` — целое число (1 ≤ m ≤ n) — количество узлов.

## Выходные данные

- Одно целое число — минимально возможная максимальная загрузка узла.

## Пример 1

**Вход:**
n = 4;
tasks = [3, 2, 5, 4]; 
m = 2

**Выход:**
7

**Пояснение:**  
Одно из оптимальных распределений процессов:

- Узел 1: [5, 2] → сумма = 7  
- Узел 2: [4, 3] → сумма = 7  

Максимальная загрузка на любом узле равна 7, что является минимально возможной.

# Краткая теория 

## Жадные алгоритмы (Greedy)

### Определение и суть

Жадные алгоритмы — это класс алгоритмов комбинаторной оптимизации, при котором на каждом шаге выбирается **локально оптимальное решение** с надеждой, что в целом это приведёт к глобально оптимальному или близкому к нему результату.  

Жадные алгоритмы обычно просты для реализации и обладают хорошей асимптотикой, но **не всегда гарантируют точный оптимум**.

### Основные свойства

1. **Локальный жадный выбор (Greedy choice)**  
   На каждом шаге алгоритм делает «жадное» решение — выбирает лучший на данный момент вариант.  

2. **Оптимальная подструктура (Optimal substructure)**  
   Решение задачи можно разбить на подзадачи, и оптимальные решения подзадач комбинируются в решение всей задачи:

$$
Opt(X) = f(Opt(X_1), Opt(X_2), \dots, Opt(X_k))
$$

3. **Отсутствие возвратов (no backtracking)**  
   Жадные алгоритмы строят решение пошагово, не перебирая все варианты.

### Примеры классических задач

- **Задача с монетами** — выдача минимального числа монет при условии, что номиналы делятся друг на друга.  
- **Рюкзак с делимыми предметами** — предметы берутся частично в порядке убывания плотности стоимости к весу.  
- **Выбор активностей** — максимизация числа непересекающихся задач, выбирая отрезки по возрастанию времени окончания.

### Преимущества и ограничения

**Преимущества:**

- Простая структура и реализация  
- Хорошая производительность: часто \(O(n \log n)\) за счёт сортировки

**Ограничения:**

- Не всегда даёт оптимальное решение  
- Требует доказательства корректности для каждой конкретной жадной стратегии

### Итог

Жадные алгоритмы — мощный инструмент для построения решений **шаг за шагом**, когда локальный оптимум ведёт к глобальному.
