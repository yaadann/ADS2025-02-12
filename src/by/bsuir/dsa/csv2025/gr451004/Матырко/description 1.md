# Название

Построение и анализ бинарного дерева поиска по списку команд

# Описание

Разработайте программу на Java, которая получает на вход одну строку, содержащую последовательность команд. Каждая команда выполняет определённое действие над бинарным деревом поиска (Binary Search Tree, BST). Команды идут подряд в одной строке и разделяются пробелами. Программа должна последовательно обрабатывать команды и выводить результаты только для тех команд, которые требуют вывода.
Допустимые команды:
Команда add X вставляет в дерево число X.
Команда del X удаляет из дерева число X, если оно присутствует.
Команда find X проверяет, существует ли в дереве число X, и выводит true или false.
Команда min находит и выводит минимальное значение дерева.
Команда max находит и выводит максимальное значение дерева.
Команда sum вычисляет сумму всех значений в дереве и выводит её. Если дерево пусто, выводится 0.
Дерево строится по стандартным правилам бинарного дерева поиска: если значение меньше значения узла — оно добавляется в левое поддерево, если больше или равно — в правое. При удалении необходимо корректно обрабатывать все случаи: удаление листа, удаление узла с одним потомком и удаление узла с двумя потомками.
Программа должна вывести результаты только тех команд, которые предполагают вывод (find, min, max, sum), причём в точной последовательности появления этих команд во входной строке.
Пример:
Ввод:
add 8 add 4 add 12 add 4 sum del 4 sum
Вывод:
28
24

# Краткая теория

Бинарное дерево поиска (Binary Search Tree, BST) — это структура данных, в которой каждый узел содержит значение и ссылки на левое и правое поддерево. Для любого узла справедливо правило: все значения в левом поддереве меньше значения узла, а все значения в правом поддереве больше или равны значению узла. Благодаря этому свойству операции поиска, вставки и удаления могут выполняться эффективно. Вставка нового значения происходит путём последовательного спуска от корня к листу: если значение меньше текущего узла, переход происходит влево, иначе — вправо, до тех пор, пока не будет найдено пустое место. Поиск выполняется аналогичным образом: каждое сравнение направляет движение влево или вправо. Удаление узла требует учёта трёх случаев: если узел является листом, его можно просто удалить; если у узла один потомок, его заменяют этим потомком; если у узла два потомка, его заменяют либо минимальным элементом правого поддерева, либо максимальным элементом левого поддерева. Минимальное значение в дереве находится переходом по левым ссылкам от корня, максимальное — по правым. Сумма всех узлов вычисляется обходом дерева, чаще всего рекурсивным.
