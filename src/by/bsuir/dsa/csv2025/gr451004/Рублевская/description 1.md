# Название

Поиск выхода

# Описание

Поиск выхода

Дан лабиринт размером `N×M`, представленный в виде матрицы. Каждая клетка может быть:

- `0` - свободный проход
- `1` - стена
- `2` - начальная позиция
- `3` - выход

Найти минимальное количество шагов от стартовой позиции до выхода. Если путь невозможен, вернуть `-1`.

### Input Format
Первая строка содержит два целых числа N и M - размеры лабиринта.
Следующие N строк содержат M целых чисел каждая, представляющих сетку лабиринта.

### Output Format
Одно число:
- Минимальное количество шагов, если путь существует
- -1, если путь невозможен

### Constraints
1 ≤ N, M ≤ 1000
Все входные значения - целые числа
Ожидаемая сложность: O(N×M)

# Краткая теория

Поиск в ширину (BFS)

Поиск в ширину — один из основных алгоритмов на графах, позволяющий находить кратчайшие пути в невзвешенных графах и решать множество других задач.

Алгоритм посещает вершины графа "волнами" — на каждом шаге обрабатываются все вершины, находящиеся на одинаковом расстоянии от стартовой вершины.

## Принцип работы
1. Инициализация: начинаем со стартовой вершины, расстояние до неё 0
2. Очередь: используем очередь для хранения вершин для обработки
3. Обход: пока очередь не пуста:
   - Извлекаем вершину из начала очереди
   - Добавляем всех её непосещённых соседей в конец очереди
   - Обновляем расстояния до соседей

## Ключевые свойства
- Гарантирует нахождение кратчайшего пути в невзвешенном графе
- Работает за O(V + E), где V — число вершин, E — число рёбер
- Использует очередь для обработки вершин в правильном порядке

BFS особенно эффективен для задач на сетках и лабиринтах, где граф представлен неявно через координаты клеток.
