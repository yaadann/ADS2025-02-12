# Название

Сжатие координат

# Описание

Сжатие координат

## Формальная постановка
Дан массив целых чисел. Необходимо преобразовать его в массив индексов, где каждое число заменяется на его позицию в отсортированном массиве уникальных элементов.

## Реальная мотивация
Сжатие координат часто используется в задачах обработки данных, когда исходные значения имеют большой разброс, но количество уникальных значений ограничено. Это позволяет эффективно использовать структуры данных, требующие непрерывной нумерации индексов.

## Формат ввода
Первая строка: целое число n (0 ≤ n ≤ 1000) - количество элементов  
Вторая строка: n целых чисел, разделенных пробелами

## Формат вывода
Массив из n целых чисел - сжатые координаты (индексация с 0)

## Пример
Ввод:
5\
5 10 2 5 2\
Вывод:\
1 2 0 1 0\
Объяснение:  
Уникальные отсортированные элементы: [2, 5, 10]  
Индексы: 2→0, 5→1, 10→2  
Исходный массив: [5, 10, 2, 5, 2] → [1, 2, 0, 1, 0]

# Краткая теория

Алгоритм сжатия координат

## Математическая формализация
Для массива $a$ длины $n$ определим:

- $b = \text{sort}(\text{unique}(a))$ - отсортированный массив уникальных элементов
- Для каждого элемента $a_i$ находим индекс в $b$: 
  $$ \text{index}(a_i) = \text{position}(a_i \in b) $$

## Алгоритм
1. **Сортировка и удаление дубликатов:**
   $$ b = \text{sort}(a) $$
   $$ b = \text{remove\_duplicates}(b) $$

2. **Бинарный поиск:**
   $$ \text{result}_i = \text{binary\_search}(b, a_i) $$

## Сложность
- **Время:** $O(n \log n)$ - доминирует сортировка
- **Память:** $O(n)$ - хранение копии массива

## Визуализация
Исходный: [5, 10, 2, 5, 2] \
Сортируем: [2, 2, 5, 5, 10] \
Уникальные: [2, 5, 10] \
Индексы: 2→0, 5→1, 10→2 \
Результат: [1, 2, 0, 1, 0]
