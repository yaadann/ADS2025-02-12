# Название

Хеширование

# Описание

Хеширование

Вам даны две строки s1 и s2.  
Перед проверкой строки должны быть нормализованы:  
- удалить все символы, кроме букв (`A-Z`, `a-z`),  
- привести все буквы к нижнему регистру,  
- убрать пробелы.

После нормализации необходимо определить, являются ли строки *усложнённо изоморфными*.

Строки считаются изоморфными, если выполняются оба условия:

1. Существует взаимно-однозначное отображение символов s1 → s2  
   (каждый символ первой строки должен отображаться в уникальный символ второй строки и наоборот).

2. Частотные профили строк совпадают  
   (множество количеств повторений всех символов одинаковое).

### Формат ввода
Две строки по одной на строку.

### Формат вывода
Одно число:
- 1, если строки изоморфны  
- 0, если не изоморфны

### Ограничения
- Длина строк: 1–200000  
- Разрешены любые ASCII-символы  
- Время работы: O(n)

# Краткая теория

Хеширование позволяет эффективно сопоставлять ключи и значения.
В Java структуры HashMap и HashSet обеспечивают операции put, get и contains за амортизированное O(1).

Для проверки изоморфности используется два хеша:
- отображение s1 → s2,
- обратное отображение s2 → s1 (для биективности).

Дополнительно используется хеширование частот: создаётся словарь вида
символ → количество вхождений.
Если мульти-множества частот совпадают, значит распределение символов аналогично.

Комбинация этих проверок делает задачу более строгой.
