# Описание

Анализ сигнала в реальном времени

В системе мониторинга поступает непрерывный цифровой сигнал, представленный в виде массива из \(N\) целочисленных значений. Для анализа качества сигнала и обнаружения аномалий используется метод скользящего окна шириной \(K\).

**Техническое задание:**
Для каждого временного интервала длиной \(K\) требуется определить:
- Минимальное значение сигнала на интервале (для обнаружения "провалов")
- Максимальное значение сигнала на интервале (для обнаружения "всплесков")

## Формат ввода

В первой строке:
- \(N\) — общее количество измерений сигнала
- \(K\) — ширина анализируемого окна

Во второй строке:
- \(N\) целых чисел — последовательные значения сигнала

## Формат вывода

Две строки:
1. Минимальные значения для каждого окна (\(N-K+1\) чисел)
2. Максимальные значения для каждого окна (\(N-K+1\) чисел)

## Пример

**Входные данные:**
7 3
2 1 3 4 6 3 8

**Выходные данные:**
1 1 3 3 3
3 4 6 6 8

*Пояснение:* Сигнал анализируется окнами [2,1,3]→[1,3,4]→[3,4,6]→[4,6,3]→[6,3,8]

**Ограничения:**
- \(1 \leq K \leq N \leq 10^6\)
- Значения сигнала: целые числа в диапазоне \([-10^9, 10^9]\)

**Требования к решению:** Эффективный алгоритм, работающий за \(O(N)\)

Задача поиска минимума и максимума в скользящем окне возникает во многих приложениях анализа данных и обработки сигналов, где требуется отслеживать экстремальные значения в потоке данных.

## Наивное решение

Прямолинейный подход — для каждого из $(N-K+1)$ окон находить минимум и максимум простым проходом по $K$ элементам. Это решение работает за $O(NK)$, что неприемлемо при больших $N$ и $K$.

## Эффективное решение с деком

Для решения за $O(N)$ используется структура данных **монотонная очередь** (monotonic queue), реализуемая через дек (deque).

### Идея алгоритма

**Для минимумов:**
- Поддерживаем дек индексов элементов, где элементы массива образуют неубывающую последовательность
- При добавлении нового элемента удаляем из конца дека все элементы, большие или равные текущему
- Голова дека всегда содержит минимум текущего окна
- Удаляем из головы элементы, которые вышли за границы окна

**Для максимумов:**
- Аналогично, но поддерживаем невозрастающую последовательность
- Удаляем из конца дека все элементы, меньшие или равные текущему
