# Название

Почти-палиндромная подстрока

# Описание

Почти-палиндромная подстрока

Дана строка `text` и число `k_len`. Найдите все стартовые индексы `i`, таких что подстрока длины `k_len` (`text[i..i+k_len-1]`) может стать палиндромом, если изменить **не более одного символа**. Вернуть массив индексов в порядке возрастания.

**Пример**

```text
text = "abca"
k_len = 4
Ответ: [0]

```text
text = "abcdefg"
k_len = 3
Ответ: [0,1,2,3,4]

## Ограничения
0 <= |text| <= 2e5
1 <= k_len <= |text|
Символы: нижний регистр a..z

Ожидаемое время работы: **O(n * k_len)** или оптимизированное скользящее окно

# Краткая теория

Краткая теория: Почти-палиндромная подстрока

Пусть подстрока \(S = text[i..i+m-1]\), длина \(m = k\_len\).  

Подстрока почти-палиндром, если количество несовпадений на симметричных позициях ≤ 1:  

$$
\#\{j : S[j] \neq S[m-1-j]\} \le 1
$$

**Проверка:**  
- Идём с двух концов (`left=0`, `right=m-1`)  
- Считаем количество несовпадений  
- Если несовпадений ≤ 1 → индекс подходит  

**Сложность:**  
Время: \(O(n \cdot k\_len)\), память: \(O(1)\)
