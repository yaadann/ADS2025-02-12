# Описание

Поиск пикового элемента

Дан массив целых чисел, который можно представить как "гористый рельеф". **Пиковым элементом** называется элемент, который больше своих соседей (если они существуют).

Например, в массиве `[1, 2, 3, 1]` элемент `3` является пиковым, так как он больше обоих соседей.

**Задача:** Найти индекс любого пикового элемента в массиве. Массив может содержать несколько пиков - достаточно найти любой из них.

**Формат ввода:**
- Первая строка: целое число `n` (1 ≤ n ≤ 1000) - размер массива
- Вторая строка: `n` целых чисел, разделенных пробелами - элементы массива

**Формат вывода:**
- Одно целое число - индекс любого пикового элемента (индексация с 0)

**Примечание:**
- Для крайних элементов достаточно быть больше единственного соседа
- Гарантируется, что пиковый элемент существует

**Пример 1:**
Ввод:
4
1 2 3 1
Вывод:
2
**Пример 2:**
Ввод:
5
1 3 2 4 5
Вывод:
1

# Краткая теория

## Поиск пикового элемента

### Определение

Элемент массива $arr[i]$ называется **пиковым**, если:

- Для $i = 0$: $arr[0] > arr[1]$
- Для $i = n-1$: $arr[n-1] > arr[n-2]$  
- Для $0 < i < n-1$: $arr[i] > arr[i-1]$ и $arr[i] > arr[i+1]$

### Эффективный алгоритм

Можно использовать модифицированный **бинарный поиск**:

1. **Инициализация:** 
   $$left = 0, \quad right = n-1$$

2. **Основной цикл:** пока $left < right$
   - Вычислить средний индекс: 
     $$mid = \left\lfloor \frac{left + right}{2} \right\rfloor$$
   - Если $arr[mid] > arr[mid+1]$ → пик находится слева: 
     $$right = mid$$
   - Иначе → пик находится справа: 
     $$left = mid + 1$$

3. **Результат:** $left$ (или $right$) - индекс пикового элемента

### Сложность

- **Временная сложность:** $O(\log n)$
- **Пространственная сложность:** $O(1)$
