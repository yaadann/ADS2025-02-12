# Название

Многопоточная сортировка слиянием

# Описание

## Задание
Дан массив целых чисел и количество потоков. Необходимо реализовать параллельную версию сортировки слиянием на Java.
Программа должна:
1. Разделить массив на указанное количество частей (примерно равных по размеру).
2. Отсортировать каждую часть в отдельном потоке с помощью стандартной сортировки (например, Arrays.sort).
3. Затем в основном потоке слить отсортированные части в один отсортированный массив с помощью алгоритма слияния.
4. Вывести отсортированный массив.
### Формат входных данных
- Первая строка: два целых числа `K N`, где `K` — количество элементов в массиве (1 ≤ K ≤ 10^5), `N` — количество потоков (1 ≤ N ≤ 16, N ≤ K).
- Вторая строка: `K` целых чисел — элементы массива (-10^9 ≤ a_i ≤ 10^9).
### Формат выходных данных
- Единственная строка: отсортированные элементы массива, разделённые пробелами.
> **Замечания:**
> - Для создания потоков используйте класс Thread или реализацию Runnable.
> - После запуска потоков используйте join() для ожидания их завершения.
> - Слияние частей выполняйте последовательно в основном потоке.
> - Рекомендуется:
> - Разделить массив на подмассивы.
> - В каждом потоке сортировать свой подмассив.
> - Затем многократно сливать пары отсортированных массивов.
> - Пример фрагмента кода на Java:
> ```java:disable-run
> class Sorter extends Thread {
>     int[] arr;
>     Sorter(int[] arr) { this.arr = arr; }
>     public void run() {
>         Arrays.sort(arr);
>     }
> }
> // В main: создать потоки, запустить, join, затем merge.
> ```
---
## Пример 1
- Ввод:
  `3 1`
  `3 1 2`
- Ожидаемый вывод:
  `1 2 3`
---
## Пример 2
- Ввод:
  `4 2`
  `4 3 2 1`
- Ожидаемый вывод:
  `1 2 3 4`

# Краткая теория

## Многопоточность в Java
### Определения
**Поток (Thread)** — это независимая единица выполнения в процессе. В Java потоки позволяют выполнять код параллельно.
**Runnable** — интерфейс для определения задачи, которую может выполнять поток.
**Синхронизация** — механизм для координации доступа к общим ресурсам, но в этой задаче не требуется, так как потоки работают с отдельными частями данных.
В задаче используются потоки для параллельной сортировки частей массива.
### Условия использования потоков
В Java потоки создаются путём наследования от Thread или реализации Runnable.
Для ожидания завершения потоков используется метод join().
Это позволяет распределить вычисления по нескольким ядрам процессора для ускорения.
### Алгоритм параллельной сортировки слиянием
1. Разделить массив на N частей.
2. Для каждой части создать поток, который отсортирует её (например, Arrays.sort).
3. Запустить потоки и дождаться их завершения (join).
4. Слить отсортированные части:
   - Использовать рекурсивное или итеративное слияние пар массивов.
   - Слияние двух отсортированных массивов: сравнивать элементы и копировать в новый массив.
     Полученный массив будет полностью отсортирован.
