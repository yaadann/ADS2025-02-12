# Название

Кратчайшие расстояния в невзвешенном графе (обход в ширину)

# Описание

Кратчайшие расстояния в невзвешенном графе (обход в ширину)

Дан неориентированный невзвешенный граф с \(n\) вершинами и \(m\) рёбрами.  
Вершины пронумерованы числами от \(1\) до \(n\).

Необходимо по заданной стартовой вершине \(s\) найти кратчайшее количество рёбер от вершины \(s\) до каждой другой вершины графа.  
Если вершина недостижима из \(s\), нужно вывести для неё \(-1\).

Гарантируется, что в графе нет кратных рёбер и петель.

## Формат входных данных

Первая строка содержит три целых числа  
\(n, m, s\) — количество вершин, количество рёбер и номер стартовой вершины  
\((1 \le n \le 2 \cdot 10^5,\ 0 \le m \le 2 \cdot 10^5,\ 1 \le s \le n)\).

Следующие \(m\) строк содержат по два целых числа \(u_i, v_i\)  
\((1 \le u_i, v_i \le n,\ u_i \ne v_i)\) — рёбра графа.  
Граф неориентированный, то есть каждое ребро соединяет вершины \(u_i\) и \(v_i\) в обе стороны.

## Формат выходных данных

Выведите \(n\) целых чисел — расстояния от вершины \(s\) до вершин \(1, 2, \dots, n\).  
Если вершина недостижима из \(s\), выведите для неё \(-1\).

Числа выводите в одной строке, разделяя пробелами.

## Пример

**Вход:**
4 3 1
1 2
2 3
3 4

**Выход:**
0 1 2 3

В этом примере кратчайший путь от вершины \(1\) до вершины \(4\) проходит через вершины \(2\) и \(3\) и содержит три ребра, поэтому расстояние равно \(3\).

# Краткая теория

Граф \(G = (V, E)\) состоит из множества вершин \(V\) и множества рёбер \(E\).  
Обходы графов — это алгоритмы, которые последовательно посещают вершины, начиная с заданной.

Два базовых обхода:

1. **Обход в глубину (DFS)** — уходит как можно дальше по одному пути, затем откатывается назад.
2. **Обход в ширину (BFS)** — сначала посещает все вершины на расстоянии \(1\), затем на расстоянии \(2\), и так далее.

Для поиска кратчайших расстояний в *невзвешенном* графе используется именно BFS.

### Идея BFS

1. Поместить стартовую вершину \(s\) в очередь.
2. Расстояние до \(s\) положить равным \(0\), до остальных — \(-1\) (ещё не посещены).
3. Пока очередь не пуста:
   - взять вершину \(v\) из начала очереди;
   - для каждой соседней вершины \(u\):
     - если \(u\) ещё не посещена (\(\text{dist}[u] = -1\)), то:
       - \(\text{dist}[u] = \text{dist}[v] + 1\);
       - добавить \(u\) в конец очереди.

Так как рёбра невзвешенные и каждое ребро учитывается ровно один раз, BFS гарантирует нахождение кратчайшего числа рёбер от \(s\) до каждой вершины.

Сложность по времени: \(O(n + m)\),  
где \(n = |V|\), \(m = |E|\).  
Память: \(O(n + m)\) при хранении графа в виде списков смежности.
