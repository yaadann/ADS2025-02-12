# Название

Сортировка подсчётом для строки

# Описание

Сортировка подсчётом для строки

## Задание

Дана строка `s`, состоящая из строчных букв латинского алфавита (`'a'`–`'z'`).  
Требуется отсортировать символы строки в порядке возрастания их кодов в таблице ASCII (т.е. по алфавиту) с помощью **алгоритма сортировки подсчётом**.

### Формат входных данных

- Одна строка `s` длиной `n` (`1 ≤ n ≤ 10^6`), содержащая только строчные латинские буквы.

### Формат выходных данных

- Одна строка, содержащая символы исходной строки `s`, отсортированные в алфавитном порядке.

> **Замечания:**
> - Сортировка подсчётом работает за время `O(n + k)`, где `k` — размер алфавита (здесь `k = 26`).
> - Алгоритм:
>   1. Создать массив `count` размером `k`, заполненный нулями.
>   2. Пройти по строке и увеличить счётчик для каждого символа.
>   3. Пройти по массиву `count` и вывести каждый символ столько раз, сколько он встретился.
> - Использование встроенных функций сортировки не допускается.

- Ввод:

- Вывод:

# Краткая теория

Теоретические сведения

## Сортировка подсчётом

### Определения

**Сортировка подсчётом** (Counting Sort) — это алгоритм сортировки, не основанный на сравнениях, который работает за линейное время при определённых условиях.

**Диапазон значений** — множество всех возможных значений, которые могут встречаться в сортируемых данных. В нашей задаче это 26 строчных букв латинского алфавита.

**Частотный массив** — массив, где каждый элемент хранит количество вхождений соответствующего значения в исходных данных.

### Условия применимости

Сортировка подсчётом эффективна когда:
1. Диапазон возможных значений `k` не слишком велик по сравнению с количеством элементов `n`
2. Данные являются целыми числами или могут быть отображены в целые числа

В нашей задаче:
- Диапазон значений: 26 букв (`k = 26`)
- Количество элементов: до 10^6 (`n ≤ 10^6`)
- Соотношение `k ≪ n`, что делает алгоритм оптимальным

### Алгоритм сортировки подсчётом

#### Шаг 1: Инициализация частотного массива
Создаём массив `count` размером 26 (для букв 'a'-'z'), заполненный нулями.

#### Шаг 2: Подсчёт частот
Проходим по всем символам входной строки и увеличиваем соответствующий счётчик

#### Шаг 3: Построение отсортированной строки
Проходим по частотному массиву и формируем результат

### Временная сложность

- **Подсчёт частот:** O(n) — один проход по строке
- **Построение результата:** O(k) — проход по алфавиту
- **Итоговая сложность:** O(n + k)

Поскольку k = 26 — константа, итоговая сложность составляет O(n).

### Преимущества и ограничения

**Преимущества:**
- Линейное время работы
- Стабильность (сохраняет порядок одинаковых элементов)
- Простота реализации

**Ограничения:**
- Требует дополнительной памяти O(k)
- Эффективен только при небольшом диапазоне значений
