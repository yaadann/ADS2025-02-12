# Название

Плохие вершины

# Описание

Плохие вершины
Дано дерево, вершины которго пронумерованы от 1 до n. Некоторые вершины являются "плохими". Задача найти количество листьев(кроме, быть может, вершины 1), путь до которых от вершины 1 не содержит m подряд идущих подряд плохих вершин. 
## Формат ввода:
В первой строке записаны два целых числа n и m (2 ≤ n ≤ 10^5, 1 ≤ m ≤ n) — количество вершин дерева и максимальное количество подряд идущих плохих вершин.

Во второй строке содержится n целых чисел a1, a2, ..., an, где каждое ai либо равняется 0 (тогда вершина i не плохая), либо равняется 1 (тогда вершина i плохая).

В следующих n - 1 строках записаны ребра дерева в формате «xi yi»(без кавычек) (1 ≤ xi, yi ≤ n, xi ≠ yi), где xi и yi — вершины дерева, соединенные очередным ребром.

Гарантируется, что данный набор рёбер задаёт дерево.
## Формат вывода
Одно целое число — ответ на задачу

# Краткая теория


### Основные понятия

#### Дерево

**Дерево** — это связный неориентированный граф без циклов. Основные свойства:

-   Между любыми двумя вершинами существует ровно один путь
    
-   Количество рёбер всегда равно $n - 1$, где $n$ — количество вершин
    
-   Является минимально связным графом (удаление любого ребра нарушает связность)
    

#### Лист

**Лист** (или висячая вершина) — вершина дерева, имеющая степень 1 (только одно инцидентное ребро). В данной задаче:

-   Листья определяются относительно всего дерева
    
-   Вершина 1 (корень) не считается листом, даже если имеет степень 1
    

#### Обход в глубину (DFS)

**Depth-First Search (DFS)** — алгоритм обхода графа, который:

-   Рекурсивно посещает "глубокие" вершины перед их соседями
    
-   Использует стек (явный или неявный) для отслеживания вершин
    
-   Время работы: $O(n + m)$ для графа с $n$ вершинами и $m$ рёбрами
