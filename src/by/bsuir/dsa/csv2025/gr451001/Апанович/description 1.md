# Название

Количество провинций

# Описание

Количество провинций

Есть `n` городов. Некоторые из них соединены, а некоторые нет. Если город `a` соединен напрямую с городом `b`, а город `b` соединен напрямую с городом `c`, то город `a` соединен косвенно с городом `c`.

**Провинция** - это группа городов, которые соединены прямо или косвенно, и никакие другие города вне этой группы не соединены с ними.

Вам дана матрица `n x n` `isConnected`, где `isConnected[i][j] = 1`, если `i-й` город и `j-й` город соединены напрямую, и `isConnected[i][j] = 0` в противном случае.

Верните общее количество провинций.

# Краткая теория

## Поиск количества провинций (компонент связности)

**Граф** — структура из вершин (городов) и рёбер (связей между ними). В задаче:
- **n вершин** — города
- **Матрица смежности** — `isConnected[i][j] = 1` означает прямое соединение

**Компонента связности** (province) — множество городов, где:
- Любые два города соединены напрямую или через другие
- Нет связей с городами вне множества

## Основные алгоритмы

### 1. Поиск в глубину (DFS)
Рекурсивно обходим все связанные города из текущего

### 2. Поиск в ширину (BFS) 
Используем очередь для обхода уровня за уровнем

### 3. DSU (Union-Find)
Объединяем связанные города в множества

**Сложность:** O(n²) времени, O(n) памяти
