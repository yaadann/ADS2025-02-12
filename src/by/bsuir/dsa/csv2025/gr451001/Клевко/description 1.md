# Название

Робин

# Описание

У Робина планируются гости — брат и мама. Нужно выбрать для каждого из них день начала визита.

Дни, доступные для визитов, пронумерованы от **1** до **n**.

У Робина и его команды назначено **k** рискованных дел.  
Каждое дело **i** занимает промежуток дней от **lᵢ** до **rᵢ** включительно.

Каждый гость остаётся у Робина **d** дней подряд. Все эти дни должны лежать внутри диапазона от **1** до **n**.

Визит считается пересекающимся с работой, если хотя бы один день визита совпадает с любым днём этой работы.


**Цель:**

- Для **брата** выбрать такой день начала визита, чтобы он пересёкся с **максимально возможным количеством работ**.
    
- Для **мамы** — такой, при котором количество пересечений будет **минимальным**.
    
- Если возможных дней несколько, необходимо выбрать **самый ранний** из них.

---
## Входные данные

- Первая строка содержит число **t** — количество тестовых наборов  
    (**1 ≤ t ≤ 10⁴**).
    

Каждый набор состоит из:

1. Строки с тремя числами:
    - **n** — общее число дней
    - **d** — длительность визита
    - **k** — количество работ  (**1 ≤ n ≤ 10⁵**, **1 ≤ d, k ≤ n**)
2. Далее идут **k** строк, каждая включает два числа **lᵢ** и **rᵢ** — границы дня начала и конца i-й работы  
    (**1 ≤ lᵢ ≤ rᵢ ≤ n**).
    

---
## Выходные данные

Для каждого набора входных данных поместите в List два целых числа, лучшие дни начала визитов брата и матери Робина соответственно. Оба визита должны укладываться между днем 1 и n включительно.

Гарантируется, что суммарное значение **n** по всем тестам не превышает **2 · 10⁵**.


# Краткая теория

Скользящее окно (Sliding window)

Используется для подсчета суммы или агрегата на подпоследовательности фиксированной длины.

Позволяет находить максимум или минимум за линейное время.

Интервальные пересечения

Часто нужно считать, сколько отрезков пересекаются с данным диапазоном.

Эффективно решается через разностный массив и скользящее окно.
