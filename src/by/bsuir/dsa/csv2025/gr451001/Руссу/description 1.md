# Название

Cумма узлов

# Описание

Сумма узлов

У инженера есть список деталей и некоторое количество металла для них, для удобства инженера из для предыдущей детали есть максимум две детали, которые можно сделать дальше, каждая деталь забирает некоторое количество металла, однако инженеру необходимо произвести готовую заготовку из деталей, потратив весь предоставленный металл, найти количество возможных заготовок

Дано бинарное дерево и число **K**.  
Необходимо найти количество ветвей (путей от корня до листа), сумма узлов которых равна **K** путь должен быть полным (заканчиваться узлом без потомков).

## Формат входных данных
1. В первой строке задано число **K**.  
2. Во второй строке задано описание бинарного дерева в формате  

root[left[left,right],right[left,right]]

## Формат выходных данных
Одно число — количество ветвей, сумма узлов которых равна **K**.

## Ограничения
- Количество узлов: не более 10^5  
- Значения узлов: от 0 до 10^4

## Теория
Бинарное дерево — это иерархическая структура данных, разновидность графа, где каждый узел имеет не более двух потомков: левый и правый.  
**Ветвь** — путь от корня дерева до листа.  
**Сумма ветви** — сумма всех значений узлов вдоль этого пути.

## Сложность

- **Время:** O(N), где N — количество узлов (каждый узел посещается один раз).  
- **Память:** O(H), где H — высота дерева (глубина рекурсии).
