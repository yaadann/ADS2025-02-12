# Название

Минимальное число монет

# Описание

Минимальное число монет

---

### Условие
Даны различные номиналы монет и целевая сумма. Найдите минимальное количество монет (можно использовать каждую монету неограниченное число раз), необходимое, чтобы получить точно эту сумму. Если получить сумму невозможно — верните `-1`.

### Формат входных данных
- Первая строка: два целых числа `n` и `amount` — количество типов монет и требуемая сумма (1 ≤ n ≤ 100, 0 ≤ amount ≤ 10^5).
- Вторая строка: `n` целых положительных чисел — номиналы монет (каждый в диапазоне 1..10^5).

### Формат выходных данных
Одно целое число — минимальное количество монет для составления `amount`, либо `-1`, если составить невозможно.

### Пример
**Ввод:**
3 11
1 2 5
**Вывод:**
3

### Пояснение
(11 = 5 + 5 +1)

# Краткая теория

Теория: Общие приёмы динамического программирования (ДП)

Динамическое программирование — метод оптимизации, применяемый, когда задача обладает двумя свойствами:

1. **Оптимальная подструктура** — решение задачи можно построить из оптимальных решений её подзадач.  
2. **Перекрывающиеся подзадачи** — одни и те же подпроблемы повторяются многократно.

---

## 1. Определение состояния (State)
В задачах типа «минимальное количество элементов для достижения суммы» удобно брать состояние:

dp[s] — минимальное число операций/элементов для получения суммы s


Состояние должно описывать подзадачу так, чтобы от неё можно было перейти к меньшим подзадачам.

---

## 2. Рекуррентное соотношение (Transition)

dp[s] = min(dp[s], dp[s - value] + 1)


Базовый принцип перехода:  
> Чтобы получить сумму s, рассмотрим каждое действие, которое ведёт в сумму s - value.

---

## 3. Инициализация (Base cases)
- `dp[0] = 0` — нулевая сумма достигается без действий.  
- `dp[s] = ∞` для остальных — сумма пока считается недостижимой.

---

## 4. Порядок вычислений
Используется **bottom-up** подход: меньшие суммы вычисляются первыми, чтобы переходы опирались на готовые значения.

---

## 5. Ключевые приёмы ДП, используемые здесь
- **Одномерное ДП** — таблица размера `amount + 1`.  
- **Минимизация по действиям** — выбор оптимального варианта.  
- **Использование "бесконечности"** — корректная обработка недостижимых состояний.  
- **Неограниченное количество действий** — каждое действие можно применять многократно.

---

## 6. Математическая интерпретация
Задача эквивалентна поиску **кратчайшего пути** в графе:

- вершины — суммы  
- ребра — действия (добавления значений)  
- вес ребра = 1  

ДП вычисляет кратчайший путь эффективно и итеративно.
