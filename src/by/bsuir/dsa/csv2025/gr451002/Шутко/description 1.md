# Название

Дерево отрезков для поиска минимума

# Описание

## Условие задачи (Дерево отрезков для поиска минимума)

### Задача: "Минимум на отрезке с обновлениями"

Реализуйте структуру данных, которая обрабатывает два типа запросов на массиве целых чисел:

1. **Запрос минимума** - `MIN l r`
   - Найти минимальный элемент в отрезке `[l, r]` (включительно)
   - Индексация начинается с 0

2. **Обновление элемента** - `UPDATE i val`
   - Установить элемент на позиции `i` равным `val`

#### Входные данные:
- Первая строка: целое число `n` - размер массива (1 ≤ n ≤ 100,000)
- Вторая строка: `n` целых чисел через пробел - элементы массива (-10^9 ≤ arr[i] ≤ 10^9)
- Третья строка: целое число `q` - количество запросов (1 ≤ q ≤ 100,000)
- Следующие `q` строк: запросы в формате:
  - `MIN l r` - запрос минимума на отрезке [l, r]
  - `UPDATE i val` - обновление элемента

#### Выходные данные:
- Для каждого запроса `MIN` выведите минимальный элемент на указанном отрезке

#### Пример:
```
Вход:
5
1 3 2 5 4
6
MIN 0 4
MIN 1 3
UPDATE 2 0
MIN 0 4
UPDATE 0 6
MIN 0 2

Выход:
1
2
0
0
```

# Краткая теория

## Теория (Дерево отрезков для поиска минимума)

### Дерево отрезков

**Дерево отрезков** — это мощная структура данных, позволяющая эффективно выполнять различные операции на отрезках массива. Основные применения включают поиск минимума, максимума, суммы на отрезке, а также обновление элементов.

#### Ключевые особенности:
- **Структура**: Бинарное дерево, где каждый узел хранит информацию об определенном отрезке массива
- **Построение**: O(n)
- **Запрос на отрезке**: O(log n)
- **Обновление элемента**: O(log n)

#### Принцип работы:
1. **Построение**: Дерево строится рекурсивно снизу вверх
2. **Запросы**: Отрезок разбивается на O(log n) предподсчитанных отрезков
3. **Обновления**: Изменения распространяются от листа к корню

#### Структура дерева:
- Корень: весь массив [0, n-1]
- Левый потомок: [0, n/2]
- Правый потомок: [n/2+1, n-1]
- Листья: отдельные элементы
