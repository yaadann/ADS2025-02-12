# Название

Оптимизация маршрутов с пересадками

# Описание

## Условие задачи

**Реальная часть:** В системе общественного транспорта города разные маршруты могут использовать разные виды транспорта (автобус, метро, трамвай). При пересадке с одного вида транспорта на другой пассажир тратит дополнительное время на ожидание следующего транспорта. Необходимо найти маршрут от начальной до конечной остановки с минимальным общим временем в пути, учитывая время пересадок.

**Формальная часть:** Дан неориентированный взвешенный граф, где каждое ребро имеет тип транспорта. При переходе по ребру с типом транспорта, отличным от предыдущего, к общему времени добавляется штраф (время пересадки для целевого транспорта). Найти кратчайший путь между двумя вершинами с учётом штрафов за пересадки.

## Формат ввода

**Ввод построчно:**
1. `N M T` - количество остановок, маршрутов, типов транспорта
2. `T чисел` - время пересадки для каждого типа транспорта
3. `M строк` по 4 числа: `u v w type` - маршруты
4. `start end` - начальная и конечная остановка

## Пример

**Пример ввода:**
3 2 2
5 3
0 1 10 0
1 2 8 1
0 2

**Разбор ввода:**
- `3 2 2` - 3 остановки, 2 маршрута, 2 типа транспорта
- `5 3` - время пересадки: 5 минут для типа 0, 3 минуты для типа 1
- `0 1 10 0` - маршрут от остановки 0 к остановке 1 за 10 минут на транспорте типа 0
- `1 2 8 1` - маршрут от остановки 1 к остановке 2 за 8 минут на транспорте типа 1
- `0 2` - найти путь от остановки 0 до остановки 2

**Пример расчета:**
Путь: 0 → 1 → 2
- 0→1: 10 минут (транспорт типа 0)
- 1→2: 8 + 3 = 11 минут (8 минут в пути + 3 минуты пересадки на транспорт типа 1)
- **Итого: 21 минута**

# Краткая теория

### Алгоритм Дейкстры
Алгоритм Дейкстры находит кратчайшие пути от начальной вершины до всех остальных в графе с неотрицательными весами.

**Основные шаги:**
1. Присваиваем начальной вершине расстояние 0, всем остальным - бесконечность
2. Выбираем вершину с наименьшим расстоянием
3. Для всех соседей этой вершины обновляем расстояния
4. Помечаем вершину как обработанную
5. Повторяем шаги 2-4 для всех вершин

### Алгоритм Дейкстры с учётом состояний
Стандартный алгоритм Дейкстры находит кратчайшие пути в графе, но не учитывает дополнительные условия. В данной задаче нам нужно отслеживать не только вершину, но и последний использованный тип транспорта.

**Модификация:**
- Храним расстояния как `dist[vertex][last_transport]`с(многомерный массив расстояний)
- При переходе по ребру проверяем, совпадает ли тип транспорта с предыдущим `lastTransport != currentTransport`
- Если типы разные, добавляем время пересадки `transferTime[currentTransport]` для целевого транспорта
