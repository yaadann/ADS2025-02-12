# Название

Бинарная куча

# Описание

Условие задачи
Реализация и анализ двоичной кучи (Min-Heap)

Цель работы
Изучить одну из важнейших структур данных — двоичную кучу, реализовать её эффективно на языке Java, доказать асимптотическую сложность операций и продемонстрировать корректность с помощью наглядных тестов.

Требуется реализовать
1. Класс BinaryHeap — минимальная двоичная куча (min-heap), хранящая элементы типа int
2. Поддержка следующих операций:
    - void insert(int value)
    - int extractMin()           — выбрасывает NoSuchElementException, если куча пуста
    - int peekMin()             — просмотр минимума без удаления
    - size(), isEmpty()
    - toString() — красивый вывод текущего состояния кучи
3. Конструктор BinaryHeap(int[] array) — построение кучи за линейное время O(n)
4. Автоматическое увеличение внутреннего массива при переполнении
5. Демонстрационная программа (с main или тестами), показывающая чёткий ввод и вывод:
   input:  [массив элементов]
   output: [отсортированные по возрастанию значения]

Требования к эффективности
- insert и extractMin — O(log n) в худшем случае
- построение кучи из n элементов — строго O(n), а не O(n log n)
- использование не более O(n) дополнительной памяти

## Что такое куча?

**Куча (heap)** — это абстрактная структура данных, реализующая **очередь с приоритетами** (priority queue).  
Она поддерживает три основные операции:

| Операция                | Сложность   | Назначение                          |
|-------------------------|-------------|-------------------------------------|
| Найти минимум / максимум| **O(1)**    | просто вернуть корень               |
| Удалить минимум / максимум | **O(log n)**| извлечь корень и восстановить кучу  |
| Добавить элемент        | **O(log n)**| вставить в конец + всплытие         |

Благодаря этим свойствам кучи используются повсюду:
- Алгоритм Дейкстры
- Алгоритм Прима (минимальное остовное дерево)
- Сортировка кучей (Heapsort)
- Планировщики задач, A* поиск и др.

---

## Двоичная куча — как она устроена?

**Двоичная куча** — это **полное дерево**, удовлетворяющее трём условиям:

1. **Свойство кучи** (для min-heap):  
   Значение родителя ≤ значения всех его потомков  
   → минимальный элемент всегда в корне!

2. **Двоичность**: у каждой вершины не более 2 детей.

3. **Полнота**: все уровни заполнены полностью, кроме последнего — и он заполняется **слева направо**.

Благодаря этому куча **идеально хранится в массиве**!
